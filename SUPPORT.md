# Support

## Where to Start

Before opening a new request, check:

- [README.md](./README.md) for setup and project overview
- [CONTRIBUTING.md](./CONTRIBUTING.md) for contribution workflow

## Open a GitHub Issue For

- reproducible bugs
- documentation gaps
- onboarding friction in the repository
- narrowly scoped feature requests

When opening an issue, include:

- what you expected
- what actually happened
- steps to reproduce
- screenshots or logs when useful
- environment details if the issue is setup-specific

## Use the Security Process For

- vulnerabilities
- exposed secrets
- authentication or authorization concerns
- anything that should not be discussed in public

See [SECURITY.md](./SECURITY.md).

## Language

English is preferred for public documentation and issues because it keeps the repository accessible to a broader contributor base.

Portuguese is acceptable when needed. Maintainers may translate or summarize details back into English for repository consistency.

## Maintainer Scope

Support is best effort. The project may prioritize:

- fixes that improve onboarding and self-hosting
- repository clarity and documentation
- regressions in the main supported path

Not every request will become roadmap work immediately.
